# 프로젝트 등록 SEO 고도화 계획

## 목표

관리자가 시공사례 등록 시 클릭 몇 번으로 SEO 친화적 데이터를 생성.
검색엔진이 "대구 OO동 하이샤시 시공" 같은 지역+품목 조합 키워드로 개별 페이지를 색인.

## description 자동 생성 템플릿 (SEO Meta Description)

```
{지역} {건물유형}에 {품목1}, {품목2} 시공. {특이사항}. 경산창호 시공.
```

예시:

- "경산시 가일리 단독주택에 하이샤시, 알루미늄도어 시공. 현관 입구 창문 + 도어 설치. 경산창호 시공."
- "대구시 본리동 상가에 스텐 난간대 시공. 경산창호 시공."

* **글자 수 제어:** 검색엔진 노출을 위해 생성되는 디스크립션은 110~150자 내외가 되도록 입력 글자 수를 제한 및 모니터링합니다.

## DB 스키마 변경

| 컬럼       | 현재          | 변경                             | 비고                                              |
| ---------- | ------------- | -------------------------------- | ------------------------------------------------- |
| `category` | `text` (단일) | `categories text[]` (다중)       | Supabase Array 타입                               |
| —          | 없음          | `region text` (시공 지역)        | UI의 시/구와 동/리를 공백 조인해 단일 텍스트 저장 |
| —          | 없음          | `building_type text` (건물 유형) | 단독주택, 아파트 등                               |

## 셀렉터 옵션

### 지역 (UI)

셀렉트(시/구) + 텍스트 입력(동/리) -> DB 저장 시 `"시/구 동/리"`로 합쳐서 저장:

- 경산시, 대구광역시

### 건물 유형

- 아파트, 빌라, 상가, 단독주택, 공장, 기타

### 시공 품목 (다중 체크)

기존 카테고리 상수에 `알루미늄도어`를 추가하여 10개로 확장 유지:

- 하이샤시, 방충망, 유리, ABS도어, 방범창, 잡철, 알루미늄도어, 방화문, 스텐, 판넬

## SEO 자동 반영 규칙

### 1. `<title>` 구성 및 글자 수 제한 규칙

- **기본 포맷:** `{제목} — {지역} {대표품목} 시공 | 경산창호`
- **대표 품목 기준 (내부 우선순위):** 다중 카테고리 선택 시, 검색 결과창 글자 수 잘림 방지를 위해 아래 가중치가 가장 높은 **1개의 품목**만 대표로 노출하고 나머지는 제목에서 생략합니다.
  - _우선순위:_ 하이샤시 > 판넬 > 방화문/ABS도어/알루미늄도어 > 유리 > 방범창 > 방충망 > 잡철/스텐
  - _예시 (하이샤시, 방충망 선택 시):_ `주택 현관문 설치 — 경산시 가일리 하이샤시 시공 | 경산창호`

### 2. Meta 및 구조화 데이터 매핑

- `meta description`: 자동 생성 description 매핑 (150자 이내)
- `JSON-LD Service`: `categories` 각각을 itemListElement에 매핑
- `JSON-LD address`: `region`을 addressLocality에 반영
- `WorkSeries` 또는 `CreativeWork` 포트폴리오 구조화 데이터 반영 (제공자 `provider`로 경산창호 연결)

## 사용자 필터 노출 정책

- **정책:** **사용자 화면(갤러리 필터)은 기존대로 단일 선택 필터를 유지합니다.**
- **이유:** 다중 필터 선택은 모바일 UI를 복잡하게 만들고, 교집합 데이터가 없어 결과창이 텅 비는 `No Results` 문제를 야기할 수 있습니다.
- **구현:** 사용자가 하나의 필터(예: '유리')를 클릭하면, DB의 `categories` 배열 내에 해당 카테고리가 포함된 시공사례를 조회(Supabase `.cs()` 연산자 이용)하여 사용자에게 매끄럽게 노출합니다.

## 작업 순서

1. DB 마이그레이션 (category → categories, region, building_type 추가)
2. 타입 / Repository / Mock 수정 (Mock 레포지토리의 `.includes()` 자바스크립트 매칭 구현 포함)
3. 관리자 폼 UI (지역/건물/품목 셀렉터 + description 자동 생성 프리뷰 + 대표 카테고리 가중치 로직 적용)
4. 갤러리 필터 단일 선택에 따른 다중 카테고리 교집합 대응
5. 상세 페이지 metadata / JSON-LD 포트폴리오 스키마 및 지역 반영
6. 기존 데이터 수동 마이그레이션 SQL 실행
